# Blow Balloon
Platform: Roblox
Genre: Tycoon
Promise: Надуй шары, выигрывай питомцев, зарабатывай деньги, надуй самый большой шар
### Summary
Blow Balloon создаёт предвкушение перед каждым надуванием шара. Тебе нужно надуть шар больше остальных, но он может лопнуть в любую секунда. Выстраивай стратегию, покупай большие шары и полагайся на удачу, ведь чем больше шар ты надуешь тем ценнее монстрика ты можешь получить

# First-Time Experience
Обучение (MVP)

4. Снарядить шар
5. Надуть шар и получить питомца
6. Поставить питомца на стенд
7. Цель: выбейте ещё двух питомцев
8. Соберите деньги со стенда
9. Цель: Накопить 1000 монет
10. Купить Розовый шар

-> Когда шар лопнет, показать сообщение: “Упс, похоже что шар может лопнуть если его сильно надуть”

luckLevel 
LuckLevel переменная которая определяет степень удачи игрока, которая влияет на то, какой питомец может выпасть игроку во время срабатывания рулетки. luckLevel состоит из baseLuck и bonuseLuck
baseLuck - По умолчанию равен 1, может увеличиться только за счёт Rebirth
bonuseLuck:
- Игрок купил временное усиление удачи
- Игрок купил постоянное усиление удачи (GamePass)
- На сервере действует усиление удачи для всех игроков
- А так же определяется за счёт степени надутости шара в момент надувания шара.

RaritySystem
Отвечает за то какой питомец выпадет игроку во время надувания шара.
- Система получает maxRarity и playerLuck (после того как игрок отпустил кнопку для надувания шара)
- maxRarity указывает на то какая максимальная редкость питомца может выпасть
- playerLuck считывает вероятность выпадения наилучшего предмета из доступных 
- Игрок может вместо питомца выбить золотой ключ. Очень редкое событие. Чем ближе игрок надул шар до своего максимума, тем с большей вероятностью выпадет ключ
- Бомба. Вероятность выпадения её всегда есть, она не зависит от степени надува шара.

# Надувание Шара

User Flow
-> Игрок подбегает к шару. Появляются кнопки действия 
	-> Игрок зажимает кнопку. Шар начинает надуваться, но может лопнуть в определённый момент 
		-> если игрок вовремя отпускает шар
			-> получает деньги
			-> если запустится рулетка с питомцами
			-> игрок получит случайного питомца
				-> игрок может поставить питомца на заработок денег на базе
		-> появляется новый шар
		-> игрок может надуть ещё один шар
		-> игрок может выйти
	-> если шар лопается	
		-> игрок теряет шар
		-> появляется новый шар
		-> игрок может надуть ещё один шар
		-> игрок может Выйти

Дефолтный шар
- По дефолту находится на стартовой локации
- Он не числится в инвентаре игрока
- Его можно использовать бесконечно

Кнопки действия
Кнопки появляются только тогда когда игрок находится в диапазоне, недалеко от шара
1. Кнопка Старт для надувания находится в HUD игрока и имеет 3 состояния. 
	- Idle. Она появилась когда игрок находится поблизости с шаром
	- Активна. Когда игрок удерживает её
	- Неактивна. Когда шар лопнул и её нельзя нажать
2. Вертикальный список с доступным шарами
	- Активные шар подсвечивается
	- Экземпляр Model активного шара заменяет предыдущий 
	- Отображаются только те шары которые имеются у игрока
3. Выйти.
	- Появляется после завершения первого надувания шара когда камера сфокусирована на шаре, игрок в этот момент не имеет управление над персонажем
	- Переключает режим камеры и возвращает игроку контроль над персонажем

Надувание шара
- По мере надувания шара растёт выигрышная сумма денег и удача
- Когда игрок отпустит шар, то он моментально получит деньги
- Если кнопку держать слишком долго шар может лопнуть, игрок не получит деньги
- Рулетка питомцев. Запускается с вероятностью 1 на 10 шаров, в случае если игрок отпустил шар. Накопленная удача влияет на результат рулетки (у всех видов шаров это значение одинаковое)
- Чем реже шар, тем больше денег шар накопит при надувании
- Шар может надуваться до 1 минуты. Но вероятность что он лопнет в первые 5 секунд уже велика (у всех видов шаров это значение одинаковое)
- Перед камерой на фоне шара находится BillboardGUI, он отображает количество денег которое постоянно увеличивается при надувании
- В нижней части экрана в ScreenGUI расположен уровень удачи luckLevel.
- Пока игрок удерживает кнопку Model шар надувается импульсивно, воспроизводится соответствующий SFX при импульсивном надувании
- 3D поведение. В исходном состоянии Model шара маленький и он лежит на земле. На начальном этапе надувания он поднимается своей макушкой вверх, затем начинает медленно подниматься до определённого уровня по мере надувания. 

Шар лопается
- Происходит тогда когда игрок не отпустил вовремя кнопку
- Игрок не получает деньги
- 3D Модель шара пропадает
- Воспроизводится характерный звук и VFX
- Изменяется состояние кнопки для надувания
- Через 1 секунду появляется такой же шар из инвентаря игрока. Если его нету, то появляется дефолтный шар
- Появление шара происходит через Tween Transparency


Камера
- Вид камеры устанавливается на стартовую позицию когда игрок нажимает кнопку старт
- Стартовая позиция. Камера фокусируется на шар, игрок отгружается (только на клиенте)
- Вид камеры возвращается обратно, когда игрок нажимает кнопку выйти. Игрок рендерится вновь
- Камера всегда смотрит на центра шара
- По мере надувания шара камера прослежиаает за ним по оси Y
- После лопания или успешного надувания шара камера плавно возвращается на стартовую позицию

Особенности архитектуры
- Для смены свойств объектов с помощью TweenService, использовать модуль `StatefulObjectController` 
- Для воспроизведения любых звуков использовать модуль AudioController.Play(SoundName)
- Сервер проверяет есть ли шар у игрока 
- Сервер определяет когда лопнет шар, если игрок вовремя не отпустит кнопку. Использовать BalloonConfig.
- Сервер определяет какой питомец выпадет

# Рулетка питомцев

UserFlow
-> Игрок отпустил шар
-> Запускается рулетка, где один питомец сменяется другим
-> Со временем скорость рулетки замедляется и останавливается 
-> Игрок нажимает кнопку забрать
-> появляется новый шар
-> игрок может надуть ещё один шар
-> игрок может Выйти


Элементы рулетки питомцев
- Птомец
- Золотой ключ (Вероятность выпадения 1/100)
- Бомба (Вероятность выпадения (1/10)

Жизненный цикл рулетки
- Рулетка появляется плавно через TweenServie свойство Transparency. Но свою работу она начинает моментально
- Продолжительность рулетки не константное. Оно может длиться в диапазоне 6-6.5 секунд.
 Возможные предметы выигрыша соответствуют уровню шара и ниже
- Рулетка стартует быстро, потом она замедляется
- Во время прокрутки питомцев они все видимые, сверху расположено название редкости с соответствующим цветом. Например Common, Rare, Epic
- Элементы рулетки никуда не перемещаются они просто заменяют друг друга
- Анимация элементов рулетки осуществляется через Scale
- Справой стороны от питомца находится бар, который указывает на то, в какой именно момент произойдёт смена. Когда бар опустится до 0, произойдет смена элемента рулетки

Один прокрут рулетки
-> Tick 1. Frame первого питомца маленькое, справа от него находится зелёный бар
-> Tick 2. Frame питомца увеличивается, Зелёный заполнитель бара уменьшается
-> Tick 3. Бар полностью пустой. Позади Frame текущего питомца, появляется Frame нового питомца. Текущий питомец начинает схлопываться в Scale 0
-> Tick 4. Старое Frame достигает Scale 0. Воспроизводится звук. Меняет своё состояние для следующего питомца с помощью TweenService. Эта анимация не зависит от общего Heardbeat системы

Момент получения питомца
- Бар мякго изчезает (становится прозрачным и исчезает). 
- Полученный питомец увеличивается 
- Подсвечивается 
- На фоне воспроизводится VFX эффек 
- Воспроизводится SFX эффект 
- Сверху появляется название с редкостью (с определённым цветом)
- Снизу 2 кнопки, Забрать и Выйти
- Model Приобретённого питомеца появляется радом с шаром и направляется в сторону базы игрока.
- Питомцы не хранятся в инвентаре игрока. Если игрок выбил питомца а у него нету места на базе, то он просто выходит на карту, ждет 30 секунд, и направляется за её пределы в определённом направлении. Другие игроки могут его забрать за донат пока тот не скрылся.
- Если выпадает золотой ключ или бомба, то они воспроизводят другие SFX и VFX эффекты, названия и кнопки действия тоже появляются

Как устроена рулетка
- Клиент получает от сервера выпадающего питомца. 
- Клиент выстраивает очередь элементов в том количестве, чтобы она прошла через всю рулетку не менее чем за 6 секунд.
- Устанавливает случайный процент, в какой именно момент остановится рулетка
- Из полученного результата получить общее время прокрутки рулетки
- Вычислить скорость с последовательным замедлением, чтобы рулетка остановилось в нужном месте
- Рендеринг происходит через Heartbeat на каждом кадре

Открытые вопросы
- Во время быстрой смены состояния Frame рендеринг его дочерних элементов или изображений может не успевать, игрок может видеть пробелы вместо питомцев во время прокрутки рулетки. Стоит ли добавить очередь Frames, или это будет оверинжинирингом?

# Progression

User Flow
-> Накопил деньги
	-> Купить новый шар из доступных
-> Получил максимальный доход от шара
	-> Поднялся в таблице лидеров
-> Накопить ключи
	-> Купить кейс для скина
	-> Купить анимацию для персонажа
	-> Повысить уровень базы
		-> На базе игрока появляется новый слот для питомца
		-> Если на этаже все слоты созданы, то появляется новый этаж

Магазин шаров
- В магазине продаются разные виды шаров.
- Вид шара определяет, какая редкость питомцев будет присутствовать в рулетке питомцев (от Common до текущий вид шара)

Rebirth
Rebirth сбрасывает часть прогресса игрока и умножает его постоянную удачу (baseLuck)
Требоавния для Rebirth:
- Определённая сумма денег
- Игрок должен иметь при наличии 2-х определённых питомцев
Актвация Rebirth обнуляет:
- Имеющихся питомцев
- Деньги
- Шары в инвентаре
Игрок получает:
- Множитель удачи
- Стартовую сумму денег (Если купил GamePass)

Развитие базы
На базе игрока находятся питомцы, они стоят в определённом закреплённом месте и приносят доход игроку. 
- Чтобы собрать доход игроку от определённого питомца нужно наступить на кнопку рядом с питомцем
- Игрок может продать определённого питомца и освободить стенд (через ProximyPrompt)
- Расширение базы добавляет этажи и дополняет новые слоты для питомцев.
- На одном этаже 10 слотов
- Каждое улучшение добавляет один слот и следующая стоимость улучшения растёт
- Цена указана в золотых ключах
- Покупка производится через SurfaceGui рядом с базой

# Monetization

Balloon Cosmetics
GamePass. У каждого шара может быть своя наклейка, купленная игроком в магазине.

Skip Animation
GamePass Пропустить анимацию рулетки, показывать сразу выпавшего питомца 

Server Luck
DevProd. Повысить уровень удачи для всех игроков на текущем сервере на 30 минут

Starter Pack
GamePass. Игрок получает стартовую сумму денег при каждом Rebirth

Шары с питомцами
DevProd. Игрок 100% выбьет питомца из этого шара

Приютить питомца
DevProd. Игрок может подойти к питомцу, у которого не хватило места на базе и выкупить его за Robux

Save Pet
GamePass. Если игрок выбивает питомца, а у него нету места на базе, то питомец выходя на общее поле, становится прозрачным и игроки не могут его забрать













More Features
Поставки шаров в магазин (ARPDAU)
- В магазине продажа шаров ограничено. Каждые 15 минут поступают новые шары в продажу

Нить от шара
Между шаром и игроком находится гибкая нить состоящая из звеньев. Одна сторона нити притягивается к игроку когда он подбегает к шару (и отвязывается когда игрок отбегает от шара). Другая сторона прикреплена к шару и поднимается за ним, когда тот взлетает.

Золотой шар с эффектом перетягивания (Progress Loop)
Из этого шара можно выбить питомца только из коллекции Gold.

Алмазный шар (Progress Loop)
- Из этого шара можно выбить питомца из коллекции Diamond

Космический шар(ARPPU)
Донатный шар из него выпадают только Легкндарные предметы из всех возможных коллекций. RaritySystem исключает возможно выпадения бомбы

Гонка рейтинга (Event)
За сжатый срок игрок должен набрать больше рейтинга чем остальные
- Используется уникальный шар
- Очки зарабатываются с этого шара

Index (+ Retention)
Отображает всех питомцев которые игрок когда-либо имел


Rebirth Save (+ ARPDAU)
Игрок может сохранить определенное количество шаров после Rebirth


Система лояльности (+ Session Length, Retention)
Предотвращает вероятность фрустрации игрока когда ему долгое время не выпадает ценный предмет

Система заданий
Игрок последовательно получает заданий и за их выполнения зарабатывает золотые ключи

Система рейтинга
- Игрок собирает комбинацию из удачных надуваний шара, получает за это определённый ранг.
- Ранг увеличивается постепенно, не перепрыгивая через один
- Ранг игрока отображается с правой стороны от ника игрока
- Ранг зависит от того, какой шар надувался
- Чем больше шар игрок надувает, тем больше очков прибавляется к рангу
- Если шар лопается, то очки убавляются
- Названия рангов: Common, Uncommon и тд.

Перетягивание питомца
С определённой долей вероятности рулетка запустится с двумя питомцами, которая в последний момент запустит интерактив, где игроку придётся нажимать правую или левую кнопку мыши, чтобы выбить именно того питомца который он захочет)


Условные обозначения
Питомец
Используется для простоты понимания системы. На самом деле они называются монстрики и имеют модель монстров. Но выполняют такой же функционал как питомцы в жанрах Tycoon
